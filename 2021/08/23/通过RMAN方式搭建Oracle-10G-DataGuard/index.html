通过RMAN方式搭建Oracle 10G DataGuard

通过RMAN方式搭建Oracle 10G DataGuard

如有错误之处，恳请指正。

[TOC]

步骤总结

  1. standby系统配置
  2. 打包软件并拷贝
  3. 解压后的操作
  4. 主库参数修改及备份
  5. 备库参数修改及还原
  6. 开启日志应用
  7. 问题处理

环境说明

primary：

​ version：Oracle 10g 10.2.0.3

​ instance_name：member

​ db_unique_name：MEMSTD2

​ OS：CentOS 5.6

standby:

​ version：Oracle 10g 10.2.0.3

​ instance_name：memberdg

​ db_unique_name：MEMBERDG

​ OS：CentOS 5.6

standby配置

  1. 操作系统参数调整

略。正常配置即可。

  1. 添加用户和组

groupadd oinstall

groupadd dba

useradd -g oinstall -G dba oracle

passwd oracle

  1. 修改.bash_profile

从primary拷贝过来

scp ~/.bash_profile standby:$PWD

也可手动修改

打包软件并拷贝

  1. 打包软件

tar -zcpf oracle.tar.gz /oracle

  1. 传输文件

scp oracle.tar.gz standby:/data/software

  1. 解压软件

tar -zxf oracle.tar.gz -C /oracle

  1. 权限管理

打包文件本身包含了权限，如果有问题也可以手动修改

chown -R oracle:oinstall /oracle

  1. 执行脚本

oralce用户下执行：

relink all

sh $ORACLE_HOME/install/changePerm.sh

root用户下执行：

sh $ORACLE_HOME/root.sh

主库参数修改及备份

下列操作如未特别说明，默认在主库操作

  1. 开启强制日志记录

保证DataGuard的可恢复性，强制数据库在任何状态下都必须记录日志

select force_logging from v$database;

-- 返回值为NO，则执行该语句开启
alter database force logging;
  1. 修改参数文件
ALTER SYSTEM SET LOG_ARCHIVE_CONFIG='DG_CONFIG=(MEMSTD2,MEMBERDG)';
ALTER SYSTEM SET LOG_ARCHIVE_DEST_2='SERVICE=MEMBERDG LGWR ASYNC VALID_FOR=(ONLINE_LOGFILES,PRIMARY_ROLES) DB_UNIQUE_NAME=MEMBERDG';
ALTER SYSTEM SET LOG_ARCHIVE_DEST_STATE_1=ENABLE;
ALTER SYSTEM SET LOG_ARCHIVE_DEST_STATE_2=ENABLE;
  1. 修改tnsnames.ora配置
-- 添加以下内容
MEMBER =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = primary)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = member1)
    )
  )

MEMBERDG =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (ADDRESS = (PROTOCOL = TCP)(HOST = standby)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = memberdg)
    )
  )
  1. 创建standby redo log

建议比online redo log多一组，size相同

alter database add standby logfile group 21 ('/data/memberdg/std_log_2101.log') size 500m;
alter database add standby logfile group 22 ('/data/memberdg/std_log_2201.log') size 500m;
alter database add standby logfile group 23 ('/data/memberdg/std_log_2301.log') size 500m;
alter database add standby logfile group 24 ('/data/memberdg/std_log_2401.log') size 500m;
alter database add standby logfile group 25 ('/data/memberdg/std_log_2501.log') size 500m;
alter database add standby logfile group 26 ('/data/memberdg/std_log_2601.log') size 500m;
alter database add standby logfile group 27 ('/data/memberdg/std_log_2701.log') size 500m;
  1. 使用rman备份主库
rman target /

run{
	allocate channel ch1 device type disk;
	allocate channel ch2 device type disk;
	allocate channel ch3 device type disk;
	allocate channel ch4 device type disk;
	allocate channel ch5 device type disk;
	allocate channel ch6 device type disk;
	allocate channel ch7 device type disk;
	allocate channel ch8 device type disk;
	backup DATABASE include CURRENT controlfile format '/data1/backup/member_%d_%T_%s_%p';
	release channel ch1;
	release channel ch2;
	release channel ch3;
	release channel ch4;
	release channel ch5;
	release channel ch6;
	release channel ch7;
	release channel ch8;
}
  1. 拷贝备份文件至standby

scp /data1/backup/* standby:$PWD

备库参数修改及还原

下列操作如未特别说明，默认在主库操作

  1. 参数文件配置

在primary生成pfile参数文件

create pfile = '/data1/backup/pfile.ora' from spfile;

拷贝至standby

scp /data1/backup/pfile.ora standby:/data/backup/

standby生成spfile文件

create spfile from pfile = '/data/backup/pfile.ora';

手动创建参数文件中指定的audit_file_dest、background_dump_dest、core_dump_dest、db_recovery_file_dest、user_dump_dest等目录

启动实例至nomount

startup nomount

修改备库参数

alter system set db_unique_name = memberdg scope=spfile;
alter system set log_archive_config='DG_CONFIG=(MEMSTD2,MEMBERDG)';
alter system set LOG_ARCHIVE_DEST_2='SERVICE=MEMSTD2 LGWR ASYNC VALID_FOR=(ONLINE_LOGFILES,PRIMARY_ROLE) DB_UNIQUE_NAME=MEMSTD2';
alter system set LOG_ARCHIVE_DEST_STATE_1=ENABLE;
alter system set LOG_ARCHIVE_DEST_STATE_2=DEFER;
alter system set remote_login_passwordfile='EXCLUSIVE' scope=spfile;
alter system set DB_FILE_NAME_CONVERT='/data/member/','/data/memberdg/','/data1/member/','/data/memberdg/' scope=spfile;
alter system set LOG_FILE_NAME_CONVERT='/data/member/','/data/memberdg/','/data1/member/','/data/memberdg/' scope=spfile;

shutdown

  1. 配置控制文件

primary执行

alter database create standby controlfile as '/data1/backup/ctl.bak';

scp /data1/backup/ctl.bak standby:/data/backup/

备库将该文件拷贝至对应目录并改名

将备库起到mount状态

startup mount
  1. 使用rman进行还原
rman target /

catalog start with '/data/backup/';

run{
	allocate channel ch1 device type disk;
	allocate channel ch2 device type disk;
	allocate channel ch3 device type disk;
	allocate channel ch4 device type disk;
	allocate channel ch5 device type disk;
	allocate channel ch6 device type disk;
	allocate channel ch7 device type disk;
	allocate channel ch8 device type disk;
	restore database;
	release channel ch1;
	release channel ch2;
	release channel ch3;
	release channel ch4;
	release channel ch5;
	release channel ch6;
	release channel ch7;
	release channel ch8;
}
  1. 创建standby redo log
alter database add standby logfile group 21 ('/data/memberdg/std_log_2101.log') size 500m;
alter database add standby logfile group 22 ('/data/memberdg/std_log_2201.log') size 500m;
alter database add standby logfile group 23 ('/data/memberdg/std_log_2301.log') size 500m;
alter database add standby logfile group 24 ('/data/memberdg/std_log_2401.log') size 500m;
alter database add standby logfile group 25 ('/data/memberdg/std_log_2501.log') size 500m;
alter database add standby logfile group 26 ('/data/memberdg/std_log_2601.log') size 500m;
alter database add standby logfile group 27 ('/data/memberdg/std_log_2701.log') size 500m;
  1. 修改tnsnames.ora

开启日志应用

  1. tnsping检查主备通信

  1. 备库开启日志应用
alter database recover managed standby database using current logfile disconnect from session ;
  1. 检查应用情况
-- 检查alert日志

-- 检查进程
select process,status,sequence#,block# from gv$managed_standby;

问题处理

  1. alert日志报错如下：

解决办法：

如果设置有fal_client和fal_server参数，DG会自动寻找缺失的日志，前提是日志文件没被覆盖或归档文件还在。

这里因为归档已经不在了，只能通过基于SCN的增量备份后进行恢复，具体步骤如下：

（1）确定SCN

-- 执行以下语句，哪个值小就从哪个开始增备
select min(first_nonlogged_scn) from v$datafile where first_nonlogged_scn > 0;
-- 可能不会有返回值，则以下面的返回值为开始SCN

select current_scn from v$database;

（2）主库执行增量备份

backup incremental from scn 48310199829 database format '/data1/backup/archivedlog_%U';

-- 备份控制文件
backup current controlfile for standby format '/data1/backup/controlfile.bak';

（3）恢复数据库

recover database noredo;

restore standby controlfile from '/data/backup/controlfile.bak';

（4）重新开启日志应用
